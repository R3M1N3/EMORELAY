# P1 功能差距评审（2026-06-13）

背景：MVP + P1~P9 已全部交付并通过真机验证（见 CLAUDE.md 仓库现状）。本文逐项评审
早期 P1 规划中悬置的功能差距，对照竞品形态（Flux-panel / Nyanpass / ForwardX）与
本项目「自有服务器 + NAT 节点 + 端口转发业务」的定位，给出做/缓/不做结论。

## 总览

| # | 差距项 | 现状 | 用户价值 | 成本 | 结论 |
|---|--------|------|----------|------|------|
| 1 | 连接数限制 | 仅累计连接统计，无并发上限 | 高（防单规则滥用拖垮节点） | 小中 | **立即做（P10a）** |
| 2 | 配置备份/恢复 | 规则导入导出已有；全量备份缺文档 | 高（容灾） | 小（文档）+中（全量导出） | **文档立即做；全量导出缓** |
| 3 | Agent 升级 | 面板显示 agent_version，升级靠手动重装 | 高（节点多时运维痛点） | 中大 | **下一阶段做（P10b）** |
| 4 | 多目标 LB / 故障转移 | 单 target_host:port | 中（高可用场景） | 大（schema+agent+健康检查） | 缓（P11 候选） |

## 逐项说明

### 1. 连接数限制 → 立即做
现状 `connection_count` 是累计值（`fetch_add` 不减），Agent 接受循环无上限；单个
用户的规则可无限开并发连接耗尽 1C 小节点的 fd/内存。方案：`forward_rules` 加
`max_connections`（NULL=不限），Agent 端维护活跃连接计数（进 +1 出 -1），达到上限
拒绝新 accept 的连接（直接断开）。与限速档同属「admin 管控资产」，普通用户不可自配。
「当前活跃连接数」的统计上报（现在只有累计）留作后续增强，不与本批混做。

### 2. 配置备份/恢复 → 文档先行
- 已有：规则导入导出（P2/P9，含归属/限速档/隧道名映射）。
- 缺：SQLite 全库备份的运维文档（WAL 模式下需 `sqlite3 .backup` 或停服拷贝，直接
  cp 活库有损坏风险）——一段文档即可堵住最大的容灾窟窿，立即做。
- 全量配置导出（用户/节点/设置/隧道 JSON）价值在「跨实例迁移」，但节点凭据
  （mTLS 证书/token 哈希）天然不可迁移，迁移后所有 Agent 都要重新接入，收益打折。
  缓，等真实迁移需求出现再设计。

### 3. Agent 升级 → 下一阶段（P10b）
节点多了之后手动 ssh 升级是最大运维痛点；面板已具备分发基建（`/dist/*` 二进制下发、
`agent_version` 上报）。方案轮廓：panel 下发 `upgrade` 命令（带目标版本 + sha256），
Agent 从 panel 下载新二进制 → 校验哈希 → 原子替换自身 → `exec` 重启（规则已落盘,
重启自动恢复）。不违反「Agent 不执行任意 shell」红线——全程 Rust 文件操作 + exec
自身,命令白名单新增一个固定动作。需要设计回滚（替换前留 .bak,启动失败回退）。
单独成一个原子单元做,不与小项混批。

### 4. 多目标 LB → 缓
schema（target 列表）、Agent（轮询/最少连接 + 健康检查 + 摘除恢复）、UI 三层都要动，
是隧道之后最大的单项。当前业务单目标转发为主，等有真实 LB 需求再立项，避免投机设计。

### 5. 2FA → 缓
面板公网暴露场景有价值。TOTP（RFC 6238）+ 恢复码,中等成本。建议与「会话管理/强制
下线」一起做成一个安全批次。当前已有登录 per-IP 限速 + Argon2,风险可控。

### 6. 分组 → 规模触发再做
`nodes.region` 已承担轻量分组职责（列表搜索可命中 region）。真分组（自定义标签 +
规则继承 + 按组筛选）等节点规模超过一屏（约 20+）再做。

### 7/9. DNS 重解析、ACL → 关闭
DNS：Agent 每个新连接都走系统解析（`TcpStream::connect((host, port))`），DDNS 目标
天然生效，无需重解析定时器；写入本文档作为结论记录。ACL：P7 已交付。

### 8/10. 图表增强、i18n → 不做
图表：节点 CPU/MEM/LOAD/流量 + 规则流量/连接时序已覆盖排障需求；更花哨的图表属
锦上添花。i18n：目标用户中文，全站文案抽取成本大,等社区出现真实需求再议。

## 排期结论

1. **P10a（本批）**：连接数限制 + SQLite 备份文档。
2. **P10b（下一批）**：Agent 一键升级（单独原子单元，含回滚设计）。
3. **P11 候选池**：多目标 LB、2FA + 会话管理、分组。按真实需求触发。
4. **关闭**：DNS 重解析（已支持）、ACL（已交付）、图表增强、i18n。
