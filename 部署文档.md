# EMORELAY 裸机部署文档（Debian 12/13，不使用 Docker）

本文从一台**全新的 Debian 12/13** 服务器开始，手把手完成 EMORELAY 主控端（panel-server + Web 前端）的源码编译部署，不依赖 Docker。

> **偷懒通道**：本文所有步骤已被一键脚本自动化，可直接运行
> `curl -fsSL https://raw.githubusercontent.com/Remine1337/EMORELAY/master/deploy.sh | bash`
> 并选择「systemd 裸机」模式。本文是该模式的逐步等价手册——路径、服务名、运行用户完全相同，手动部署后同样可以用脚本做升级/备份/卸载。

## 0. 最终拓扑

```
浏览器 ──HTTP/HTTPS──► Caddy (:80/:443)
                        ├── /api/*、/install.sh、/dist/*  ──反代──►  panel-server REST (127.0.0.1:8080)
                        └── 其余路径 ──静态文件──►  /var/www/emorelay (SPA 前端)

节点 Agent ──gRPC mTLS──► panel-server gRPC (0.0.0.0:50051,不经 Caddy)
```

- panel-server **不内嵌**前端静态文件服务，必须由 Caddy（或其他 Web 服务器）托管前端并反代 API。
- gRPC 50051 由 panel-server 内置 CA 的 mTLS 自保护，直接对公网监听，**不要**放进 Caddy 反代。

### 部署后的文件布局

| 路径 | 内容 |
|---|---|
| `/opt/emorelay` | 源码（git clone，升级时 `git pull` 重编译） |
| `/usr/local/bin/emorelay-panel` | 主控二进制（从构建产物复制，与构建树解耦） |
| `/var/lib/emorelay` | 数据目录：`emorelay.db`（SQLite）、`agent-dist/`（节点二进制分发）、`tls/`（内置 CA） |
| `/etc/emorelay/panel.env` | 配置文件（0600，含密钥） |
| `/etc/systemd/system/emorelay-panel.service` | systemd 服务 |
| `/var/www/emorelay` | 前端静态文件 |
| `/etc/caddy/Caddyfile` | Caddy 站点配置 |

## 1. 准备工作

### 1.1 系统要求

- Debian 12（bookworm）或 13（trixie），amd64 或 arm64，root 权限。
- **内存 + swap 合计 ≥ 3GB**（Rust 编译需要）。1GB 小鸡先建 swap：

```bash
fallocate -l 2G /swapfile-emorelay
chmod 600 /swapfile-emorelay
mkswap /swapfile-emorelay
swapon /swapfile-emorelay
echo '/swapfile-emorelay none swap sw 0 0' >> /etc/fstab
```

- 域名可选：有域名（A 记录已指向本机）则 Caddy 自动申请 HTTPS 证书；没有就用 IP + HTTP。

### 1.2 ⚠ 部署前必须想清楚的一件事：PANEL_PUBLIC_HOST

`PANEL_PUBLIC_HOST`（域名或公网 IP）会被写进 gRPC server 证书的 SAN，并且**首次启动后固化**——panel-server 检测到 `${PANEL_DATA_DIR}/tls/ca.pem` 已存在就直接复用整套证书，之后改这个变量不会重新签发。

事后更改的代价：删除 `/var/lib/emorelay/tls` 目录并重启（重新生成 CA），**所有已接入的 Agent 都要重新安装凭据**。所以开工前先定好对外地址。

## 2. 安装依赖

### 2.1 基础工具与编译依赖

```bash
apt-get update
apt-get install -y git curl ca-certificates openssl \
    build-essential pkg-config libssl-dev musl-tools
```

> 不需要装 `protobuf-compiler`：构建脚本使用 vendored protoc，装系统 protoc 反而可能引入版本冲突。

### 2.2 Rust（rustup，不能用 apt 的 rustc）

项目 MSRV 为 Rust 1.80，而 Debian 12 仓库里的 rustc 是 1.63，**不可用**。必须走 rustup：

```bash
curl -fsSL https://sh.rustup.rs | sh -s -- -y --default-toolchain stable --profile minimal
. "$HOME/.cargo/env"
```

### 2.3 Node.js 22（NodeSource）

前端构建需要 Node 20+。Debian 12 自带 18 太老；统一用 NodeSource 22：

```bash
curl -fsSL https://deb.nodesource.com/setup_22.x | bash -
apt-get install -y nodejs
node --version   # 应为 v22.x
```

> Debian 13 自带的 nodejs 是 20.x，满足要求；若 NodeSource 不可用可直接 `apt-get install -y nodejs npm`。

### 2.4 Caddy（官方 apt 源）

```bash
apt-get install -y debian-keyring debian-archive-keyring apt-transport-https
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' \
    | gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg --yes
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' \
    > /etc/apt/sources.list.d/caddy-stable.list
apt-get update
apt-get install -y caddy
```

## 3. 获取源码与编译

### 3.1 clone

```bash
git clone --depth 1 https://github.com/Remine1337/EMORELAY.git /opt/emorelay
cd /opt/emorelay
```

### 3.2 编译 panel-server

```bash
cargo build --release -p panel-server
```

首次编译约 5-15 分钟。内存吃紧的机器加 `-j 1` 降低峰值（更慢但不 OOM）。

### 3.3 编译 node-agent（musl 静态，供节点下载）

panel-server 提供节点一键安装端点 `/install.sh`，节点机会从 `/dist/node-agent-linux-<arch>` 下载 Agent 二进制——这些文件需要你放进 `${PANEL_DATA_DIR}/agent-dist/`。

**必须编 musl 静态二进制**：节点可能是 Debian 11 / Ubuntu 20.04 等旧 glibc 系统，主控机上编出的 glibc 动态二进制在旧系统会直接报 `GLIBC_x.xx not found`。

```bash
# amd64 主控:
rustup target add x86_64-unknown-linux-musl
cargo build --release -p node-agent --target x86_64-unknown-linux-musl

# arm64 主控则替换为:
# rustup target add aarch64-unknown-linux-musl
# cargo build --release -p node-agent --target aarch64-unknown-linux-musl
```

文件名必须严格是 `node-agent-linux-amd64` / `node-agent-linux-arm64`（panel-server 的 `/dist` 路由有白名单），放置步骤见 4.1。如果你的节点和主控不是同一架构，见附录 A 交叉编译。

### 3.4 构建前端

```bash
cd /opt/emorelay/web
npm ci --no-audit --no-fund
npm run build
cd /opt/emorelay
```

产物在 `/opt/emorelay/web/dist`。

## 4. 配置

### 4.1 创建用户与目录

```bash
useradd --system --no-create-home --shell /usr/sbin/nologin emorelay

# 数据目录 + agent 二进制分发目录
install -d -m 0750 /var/lib/emorelay /var/lib/emorelay/agent-dist

# 主控二进制
install -m 0755 /opt/emorelay/target/release/panel-server /usr/local/bin/emorelay-panel

# agent 二进制(amd64 主控为例;arm64 主控把两处 amd64 换成 arm64)
install -m 0755 /opt/emorelay/target/x86_64-unknown-linux-musl/release/node-agent \
    /var/lib/emorelay/agent-dist/node-agent-linux-amd64

chown -R emorelay:emorelay /var/lib/emorelay

# 前端静态文件(重装时目标目录已存在的话,cp -r 会嵌套出 emorelay/dist,先删旧目录)
mkdir -p /var/www
rm -rf /var/www/emorelay
cp -r /opt/emorelay/web/dist /var/www/emorelay
chown -R caddy:caddy /var/www/emorelay

# 配置目录
install -d -m 0750 /etc/emorelay
```

| 路径 | 属主 | 权限 |
|---|---|---|
| `/var/lib/emorelay`（递归） | emorelay:emorelay | 0750 |
| `/etc/emorelay` | root:root | 0750 |
| `/etc/emorelay/panel.env` | root:root | **0600**（含密钥；systemd 以 root 读取，emorelay 用户无需读权限） |
| `/var/www/emorelay` | caddy:caddy | 默认 |

### 4.2 /etc/emorelay/panel.env

先生成两个凭据（**记下输出**，下面会用到）：

```bash
openssl rand -hex 32        # 作为 PANEL_JWT_SECRET
openssl rand -base64 18     # 作为管理员初始密码(也可自己设)
```

写配置（把 `<...>` 占位符换成实际值；外层 `umask 077` 保证文件落盘瞬间就是 0600，不给其他用户可读窗口）：

```bash
(umask 077; cat > /etc/emorelay/panel.env <<'EOF'
RUST_LOG=info,sqlx=warn
# REST 只绑本机回环,由 Caddy 反代;gRPC 直接对外(mTLS 自保护)。
PANEL_BIND_ADDR=127.0.0.1:8080
PANEL_GRPC_BIND_ADDR=0.0.0.0:50051
PANEL_DATABASE_URL=sqlite:///var/lib/emorelay/emorelay.db
PANEL_DATA_DIR=/var/lib/emorelay
PANEL_JWT_SECRET=<上面生成的 hex64>
PANEL_JWT_EXPIRY_HOURS=24
PANEL_CORS_ORIGIN=<https://你的域名 或 http://你的IP>
PANEL_BOOTSTRAP_ADMIN_USERNAME=admin
# 首次成功登录后可清空本行(已建库即忽略),减少一处明文。
PANEL_BOOTSTRAP_ADMIN_PASSWORD=<管理员初始密码>
PANEL_PUBLIC_BASE_URL=<同 PANEL_CORS_ORIGIN>
PANEL_PUBLIC_HOST=<你的域名 或 公网IP,见 1.2 警告>
PANEL_DEV_DISABLE_MTLS=0
EOF
)
```

变量逐项说明：

| 变量 | 说明 |
|---|---|
| `PANEL_BIND_ADDR` | REST 监听地址。裸机方案只绑 `127.0.0.1`，由 Caddy 反代，比直接对外更收紧 |
| `PANEL_GRPC_BIND_ADDR` | gRPC 控制面（Agent 接入点），mTLS 强制，对外监听 |
| `PANEL_DATABASE_URL` | SQLite 库路径（启动自动建库、跑 migrations、开 WAL） |
| `PANEL_JWT_SECRET` | JWT 签名密钥。程序不强制长度，但短密钥易被暴力破解，建议 ≥32 字符随机串 |
| `PANEL_CORS_ORIGIN` | 允许的前端来源，必须与浏览器地址栏 origin 逐字符一致 |
| `PANEL_BOOTSTRAP_ADMIN_PASSWORD` | 首次启动自动创建的 admin 密码（库里已有 admin 则忽略） |
| `PANEL_PUBLIC_BASE_URL` | 用于生成节点一键安装命令里的下载链接 |
| `PANEL_PUBLIC_HOST` | gRPC 证书 SAN 的对外主机名，**首启固化**，见 1.2 |

完整变量参考见 `docs/deployment.md` 第三节。

### 4.3 systemd unit

```bash
cat > /etc/systemd/system/emorelay-panel.service <<'EOF'
[Unit]

[Service]

# 安全收紧(与 scripts/emorelay-agent.service 同风格)

ProtectHome=true
PrivateTmp=true
# SQLite 库 + WAL/SHM、内置 CA tls/、agent-dist 都在数据目录下,必须显式放行写。
ReadWritePaths=/var/lib/emorelay
ProtectKernelTunables=true
ProtectKernelModules=true
ProtectKernelLogs=true
ProtectControlGroups=true
RestrictNamespaces=true
LockPersonality=true
RestrictRealtime=true
RestrictSUIDSGID=true
MemoryDenyWriteExecute=true
SystemCallArchitectures=native
RestrictAddressFamilies=AF_INET AF_INET6 AF_UNIX

[Install]
WantedBy=multi-user.target
EOF
```

### 4.4 Caddyfile

为什么 `/install.sh` 和 `/dist/*` 要单独反代：panel-server 的节点一键安装端点不在 `/api` 前缀下（见 `crates/panel-server/src/routes/mod.rs`），漏掉它们节点机就下载不到安装脚本和 Agent 二进制。

为什么用 `handle` 而不是 `handle_path`：`handle_path` 会把匹配前缀剥掉再反代，而 panel-server 的路由本身就带 `/api` 前缀，剥掉必 404。

先备份原配置（仅首次，避免重跑时用 EMORELAY 配置覆盖真正的原始备份），再写入（**IP 直连模式**，站点行为 `:80`）：

```bash
[ -f /etc/caddy/Caddyfile.bak-emorelay ] || cp /etc/caddy/Caddyfile /etc/caddy/Caddyfile.bak-emorelay

cat > /etc/caddy/Caddyfile <<'EOF'
# EMORELAY 主控站点
:80 {
    encode gzip zstd

    # REST API(panel-server 只绑 127.0.0.1:8080)。
    handle /api/* {
        reverse_proxy 127.0.0.1:8080
    }
    # Agent 一键安装脚本与二进制分发,不在 /api 前缀下。
    handle /install.sh {
        reverse_proxy 127.0.0.1:8080
    }
    handle /dist/* {
        reverse_proxy 127.0.0.1:8080
    }

    # SPA 前端
    handle {
        root * /var/www/emorelay
        try_files {path} /index.html
        file_server
    }

    log {
        output file /var/log/caddy/emorelay.log
    }
}
EOF
```

**域名 + 自动 HTTPS 模式**：把第一行的 `:80 {` 换成你的域名（如 `panel.example.com {`），其余不变。Caddy 会自动申请/续期 Let's Encrypt 证书并把 80 跳转到 443。前提：域名 A 记录已指向本机，且 80/443 端口对外可达。

## 5. 启动与验证

### 5.1 启动

```bash
systemctl daemon-reload
systemctl enable --now emorelay-panel
systemctl enable caddy
systemctl restart caddy

# 写入部署标记:一键脚本 deploy.sh 靠它识别已安装状态,
# 写了这一步,手动部署的机器才能用脚本的升级/备份/卸载菜单接管。
printf 'systemd\n/opt/emorelay\n' > /etc/emorelay/deploy-mode
```

验证：

```bash
systemctl status emorelay-panel caddy --no-pager
curl -fsS http://127.0.0.1:8080/api/health     # 应返回 200
ss -tlnp | grep -E ':(80|443|8080|50051)\s'    # 四个端口都应在监听(无 HTTPS 则没有 443)
```

### 5.2 防火墙放行

需要对外开放：**80**（HTTP；HTTPS 模式还要 **443**）和 **50051**（Agent gRPC）。8080 不需要对外。

ufw：

```bash
ufw allow 80/tcp
ufw allow 443/tcp     # 仅 HTTPS 模式
ufw allow 50051/tcp
```

nftables：确认 input 链放行上述 TCP 端口（具体命令取决于你的规则集结构）。

### 5.3 首次登录

浏览器打开 `http://<你的IP>`（或 `https://<你的域名>`），用 `admin` + 你在 4.2 设置的初始密码登录。

登录成功后建议把 `/etc/emorelay/panel.env` 里的 `PANEL_BOOTSTRAP_ADMIN_PASSWORD` 行清空（库已建好，该变量此后被忽略），减少一处明文。

## 6. Agent 节点接入

1. 登录面板 → **系统设置**，把 Agent 控制端点设置为 `https://<PANEL_PUBLIC_HOST>:50051`（注意是 `https://`，mTLS 默认强制）。
2. **节点管理** → 新建节点，面板会生成一条一键安装命令（形如 `curl ... /install.sh | bash -s -- --token=...`），到节点机上以 root 执行即可。
3. 工作原理：节点机从 `PANEL_PUBLIC_BASE_URL/install.sh` 拿到安装脚本，再从 `/dist/node-agent-linux-<arch>` 下载你在 3.3 放进 `agent-dist/` 的二进制，写入 mTLS 凭据后注册 `emorelay-agent` systemd 服务。
4. 节点机也会用到 `/etc/emorelay`（agent.env）和 `/var/lib/emorelay`（agent-state.json）——与主控同名不同文件，主控机自己兼任节点也不冲突。

排错入口：节点上 `journalctl -u emorelay-agent -f`；mTLS 失败常见原因见 10.3。

## 7. 升级

```bash
# 0) 先备份(见第 8 节)
cd /opt/emorelay
git pull --ff-only

# 1) 重新编译三件套(命令同 3.2-3.4)
. "$HOME/.cargo/env"
cargo build --release -p panel-server
cargo build --release -p node-agent --target x86_64-unknown-linux-musl   # arm64 主控换 target
cd web && npm ci --no-audit --no-fund && npm run build && cd ..

# 2) 先构建后停服,停机窗口只有秒级
systemctl stop emorelay-panel
install -m 0755 target/release/panel-server /usr/local/bin/emorelay-panel
# arm64 主控:下面两行的 x86_64-unknown-linux-musl 换 aarch64-unknown-linux-musl,amd64 换 arm64
install -m 0755 target/x86_64-unknown-linux-musl/release/node-agent \
    /var/lib/emorelay/agent-dist/node-agent-linux-amd64
chown -R emorelay:emorelay /var/lib/emorelay

# 前端原子替换,避免窗口期 Caddy 服务到半成品
cp -r web/dist /var/www/emorelay.new
mv /var/www/emorelay /var/www/emorelay.old
mv /var/www/emorelay.new /var/www/emorelay
rm -rf /var/www/emorelay.old
chown -R caddy:caddy /var/www/emorelay

systemctl start emorelay-panel
curl -fsS http://127.0.0.1:8080/api/health
```

数据库 migrations 由 panel-server 启动时自动执行（migrations 编译期内嵌在二进制里），无需手动操作。

## 8. 备份与恢复

**备份**（短暂停服保证 SQLite WAL 一致性）：

```bash
systemctl stop emorelay-panel
tar czf /root/emorelay-backup-$(date +%Y%m%d-%H%M%S).tar.gz -C / var/lib/emorelay etc/emorelay
systemctl start emorelay-panel
```

> ⚠ 备份包内含**内置 CA 私钥**（`tls/`）与 **JWT secret / 管理员密码**（`panel.env`），务必妥善保管。

**恢复**（新机器先完成第 2-4 节，再展开备份）：

```bash
systemctl stop emorelay-panel
tar xzf /root/emorelay-backup-<时间戳>.tar.gz -C /
chown -R emorelay:emorelay /var/lib/emorelay
systemctl start emorelay-panel
```

恢复了 `tls/` 就保住了 CA——所有已接入 Agent 无需重装凭据。

## 9. 卸载

```bash
systemctl disable --now emorelay-panel
rm -f /etc/systemd/system/emorelay-panel.service
systemctl daemon-reload
rm -f /usr/local/bin/emorelay-panel
rm -rf /var/www/emorelay

# 恢复原 Caddy 配置
mv /etc/caddy/Caddyfile.bak-emorelay /etc/caddy/Caddyfile
systemctl restart caddy

# ⚠ 以下不可恢复:数据目录含 SQLite 库与内置 CA,删除后所有 Agent 失联
rm -rf /var/lib/emorelay
rm -rf /etc/emorelay
rm -rf /opt/emorelay
userdel emorelay
```

Rust / Node / Caddy 软件包不在上面清单里，需要时自行 `apt remove`。

## 10. 故障排查

### 10.1 编译 OOM（被 Killed）

`free -h` 看内存+swap 是否 ≥3GB；不够先按 1.1 建 swap，或用 `cargo build --release -p panel-server -j 1` 单线程编译。

### 10.2 服务起不来

`journalctl -u emorelay-panel -n 50 --no-pager`，常见报错对照：

| 报错特征 | 原因 |
|---|---|
| bootstrap admin 相关错误 | 首次启动未设 `PANEL_BOOTSTRAP_ADMIN_PASSWORD` |
| 数据库 open 失败 / 权限拒绝 | `/var/lib/emorelay` 属主不是 emorelay，重跑 `chown -R emorelay:emorelay /var/lib/emorelay` |
| EACCES / Read-only file system | unit 的 `ReadWritePaths` 没覆盖到写入路径（自定义了数据目录却没改 unit） |

### 10.3 Agent 连不上

- **证书 SAN 不匹配**：Agent 日志报 hostname/SAN 校验失败 → `PANEL_PUBLIC_HOST` 与 Agent 实际连接的地址不一致。代价见 1.2。
- **50051 被防火墙挡**：主控机 `ss -tlnp | grep 50051` 确认监听，再查 ufw/云厂商安全组。
- **端点协议写错**：系统设置里的控制端点必须是 `https://`（mTLS），写成 `http://` 握手必失败。

### 10.4 浏览器报 CORS 错误

`PANEL_CORS_ORIGIN` 必须与地址栏 origin **逐字符一致**（协议、域名、端口都算）。改完 `systemctl restart emorelay-panel`。

### 10.5 Debian 12 与 13 的差异

仅两点影响本文：Debian 12 自带 Node 18（必须走 NodeSource），13 自带 20（可直接 apt 装）；其余步骤两版完全一致。

## 附录 A：交叉编译另一架构的 node-agent

主控是 amd64 但有 arm64 节点（或反之）时，需要补编另一架构的 musl 静态二进制：

```bash
# amd64 主控编 arm64 agent:
apt-get install -y gcc-aarch64-linux-gnu
rustup target add aarch64-unknown-linux-musl
CARGO_TARGET_AARCH64_UNKNOWN_LINUX_MUSL_LINKER=aarch64-linux-gnu-gcc \
CC_aarch64_unknown_linux_musl=aarch64-linux-gnu-gcc \
    cargo build --release -p node-agent --target aarch64-unknown-linux-musl
install -m 0755 target/aarch64-unknown-linux-musl/release/node-agent \
    /var/lib/emorelay/agent-dist/node-agent-linux-arm64
chown emorelay:emorelay /var/lib/emorelay/agent-dist/node-agent-linux-arm64
```

arm64 主控编 amd64 agent 同理：包名换 `gcc-x86-64-linux-gnu`，target 换 `x86_64-unknown-linux-musl`，环境变量换 `CARGO_TARGET_X86_64_UNKNOWN_LINUX_MUSL_LINKER=x86_64-linux-gnu-gcc` 与 `CC_x86_64_unknown_linux_musl=x86_64-linux-gnu-gcc`，产物文件名 `node-agent-linux-amd64`。

> 注：交叉编译需同时给 `CARGO_TARGET_*_LINKER` 和 `CC_*`（ring 等含 C 代码的依赖要用目标 CC）。也可用一键脚本菜单里的「补编另一架构 agent」自动完成。
